# Darwin's Canvas Agent Rules

## UI Consistency
**CRITICAL RULE**: Do not change the overall aesthetic, style, or color scheme of UI components unless explicitly directed to do so by the user. 
Always refer to `docs/design.md` and `docs/style-guide.md` before making any visual changes.
The established aesthetic (Playful Clean Design Tokens, soft colors, specific drop-shadows, non-glassmorphic styling) must be strictly maintained. When fixing UX issues, localize your changes narrowly (like adding a border to a specific cursor) rather than overhauling entire CSS blocks or themes.
